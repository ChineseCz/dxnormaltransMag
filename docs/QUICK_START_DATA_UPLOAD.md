## 场景一：电抗器参数化数据（最简单）

**你的数据长这样**：
```
电抗器/
  ├─ 100[A].txt     ← 电流100A时的磁场分布
  ├─ 200[A].txt     ← 电流200A时的磁场分布
  ├─ 300[A].txt
  └─ ...
```

**每个文件内容**（2D轴对称，r-z坐标）：
```
% Dimension: 2
% Nodes: 91462
% r         z         Color
0.1        0.5       0.025
0.15       0.5       0.032
...
```

### 操作步骤

#### Step 1：创建数据集
1. 点击"数据集管理" → "新建数据集"
2. 填写信息：
   - 名称：`电抗器磁场-参数化扫描`
   - 设备类型：电抗器
   - 物理场：磁场
   - **坐标系**：`rz` ⚠️ 关键！
   - **数据组织方式**：`逐工况分文件 (perfile)` ⚠️ 关键！
   - 坐标列数：`2`
3. 输入变量：
   ```
   名称: 励磁电流
   单位: A
   符号: I
   ```
4. 输出变量：
   ```
   名称: 磁通密度
   单位: T
   符号: B
   ```
5. 点击"创建"

#### Step 2：批量上传文件
1. 进入刚创建的数据集 → "文件上传"
2. **选择所有 .txt 文件**（可多选）
3. 上传时系统会自动：
   - 识别文件角色为 `output`
   - 从文件名提取工况值（100, 200, 300, ...）
   - 显示：`100[A].txt → 工况值: 100.0 A`
4. 如果文件名不规范，可手动填写"工况值"

#### Step 3：数据处理（一键完成）
1. 进入"数据处理"
2. **Step 1：装配**
   - 点击"执行"，系统自动按工况值排序装配
   - 输出：`100个工况 × 91462个空间测点`
3. **Step 2：划分**
   - 训练集比例：80%（默认）
   - 点击"执行" → 输出：训练80条 / 测试20条
4. **Step 3：归一化**
   - 点击"执行"（无需参数）
5. **Step 4：PCA降维**
   - 主成分数：60（默认）
   - 点击"执行" → 输出：91462维 → 60维

✅ **完成！** 现在可以进入"模型训练"页面开始训练神经网络。

## 场景二：单相变压器时域数据

**你的数据长这样**：
```
单相变压器磁场/raw data/
  ├─ mag-100-10k-50-0.2-0.0005.txt      ← 磁场输出（1241点 × 400时间步）
  ├─ primcur-100-10k-50-0.2-0.0005.txt  ← 初级电流（时间序列）
  └─ seccur-100-10k-50-0.2-0.0005.txt   ← 次级电流（时间序列）
```

**磁场文件格式**（3D xyz坐标 + 400时间步）：
```
% Dimension: 3
% Nodes: 1241
% x      y      z      mf.normB@t=0   mf.normB@t=0.0005  ...  mf.normB@t=0.2
0.1     0.2    0.0    0.035          0.042              ...  0.038
0.15    0.2    0.0    0.051          0.058              ...  0.055
```

**电流文件格式**（无空间维度，纯时间序列）：
```
% Dimension: 0
% Nodes: 1
% t(s)    mf.ICoil_1(A)
0.0       0.0
0.0005    12.5
0.001     24.8
...
```

### 操作步骤

#### Step 1：创建数据集
- 名称：`单相变压器-50Hz暂态磁场`
- 设备类型：变压器
- 物理场：磁场
- **坐标系**：`xyz`
- **数据组织方式**：`单文件多工况 (multicolumn)`
- 坐标列数：`3`
- **时间步长**：`0.0005` ⚠️ 用于后续截取
- 输入变量：
  ```
  [
    {名称: "初级电流", 单位: "A", 符号: "I1"},
    {名称: "次级电流", 单位: "A", 符号: "I2"}
  ]
  ```
- 输出变量：`{名称: "磁通密度", 单位: "T", 符号: "B"}`

#### Step 2：上传文件（注意顺序）
1. 上传 `mag-*.txt`，角色：`output`
2. 上传 `primcur-*.txt`，角色：`input`，变量索引：`0`
3. 上传 `seccur-*.txt`，角色：`input`，变量索引：`1`

#### Step 3：自动检测稳态起始点（可选）
1. 点击"自动检测稳态"
2. 系统分析电流波形，建议 t0=0.04s

#### Step 4：数据处理
1. **Step 1：截取稳态**
   - 起始时间 t0：`0.04` s
   - 结束时间 tEnd：`0.1` s
   - 时间步长 dt：`0.0005` s
   - 点击"执行" → 输出：120个时间步 × 1241个空间点
2. 后续步骤同场景一

## 场景三：变压器温升数据（多参数）

**你的数据长这样**：
```
变压器温升计算/导出数据/
  ├─ 1158.6_865.8.txt   ← 文件名包含2个参数（P1=1158.6W, P2=865.8W）
  ├─ 1190.7_889.85.txt
  └─ ...
```

**文件格式**（2D rz坐标 + 温度场）：
```
% Dimension: 2
% Nodes: 4286
% r         z         Color
0.1        0.5       313.8
0.15       0.5       314.2
```

### 操作步骤

#### Step 1：创建数据集
- 名称：`变压器温升场-双功率参数`
- 设备类型：变压器
- 物理场：温度场
- **坐标系**：`rz`
- **数据组织方式**：`逐工况分文件 (perfile)`
- 坐标列数：`2`
- 输入变量：
  ```
  名称: 总功率  ← 暂时只取第一个参数（未来版本支持多参数）
  单位: W
  符号: P
  ```

#### Step 2：上传 + 手动设置工况值
由于文件名格式特殊（`P1_P2.txt`），需要手动处理：

**方案 A：批量重命名文件（推荐）**
```python
# rename_files.py
import os
import re

folder = "E:/path/to/导出数据"
for fname in os.listdir(folder):
    m = re.search(r'^([\d.]+)_[\d.]+\.txt', fname)
    if m:
        new_name = f"{m.group(1)}[W].txt"
        os.rename(os.path.join(folder, fname), 
                  os.path.join(folder, new_name))
print("重命名完成！")
```
运行后文件变为：`1158.6[W].txt`, `1190.7[W].txt`, ...  
然后正常上传，系统自动识别工况值。

**方案 B：上传后手动编辑**
1. 批量上传原始文件
2. 在文件列表中逐个点击"编辑"
3. 手动填写"工况值"（如 1158.6）

#### Step 3：数据处理
同场景一

## 场景四：三相变压器（多通道输入）⚠️ 高级

**数据结构**：
```
三相变压器/raw data/
  ├─ H1.txt         ← 3个线圈电流（Dimension=0，243列=3通道×81时间步）
  ├─ 三相1.0.txt    ← 磁场输出（Dimension=3，正常时域数据）
```

**输入文件特殊格式**（多通道交织）：
```
% Dimension: 0
% Nodes: 1
% mf.ICoil_1@t=1.16  mf.ICoil_4@t=1.16  mf.ICoil_7@t=1.16  mf.ICoil_1@t=1.1605  ...
-7.046                -55.030             60.880             3.349                ...
```
数据排列：[I1@t0, I4@t0, I7@t0, I1@t1, I4@t1, I7@t1, ...]

### 操作步骤

⚠️ **当前版本限制**：系统仅支持单通道输入，多通道数据会被简化为第一通道。  
**解决方案**：
1. **临时方案**：手动拆分文件（用 Python 脚本）
2. **等待更新**：v1.1 版本将支持多通道自动解交织

**Python 拆分脚本示例**：
```python
import numpy as np

# 读取原始文件
data = np.loadtxt('H1.txt', encoding='utf-8', comments='%')
# data shape: (1, 243) = (1, 3通道×81时间步)

# 重组为 (81, 3)
n_channels = 3
n_timesteps = data.shape[1] // n_channels
reshaped = data.reshape(n_timesteps, n_channels)

# 保存为3个独立文件
np.savetxt('ICoil_1.txt', reshaped[:, 0])
np.savetxt('ICoil_4.txt', reshaped[:, 1])
np.savetxt('ICoil_7.txt', reshaped[:, 2])
```

然后按 multicolumn 模式上传3个输入文件，variableIndex 分别设为 0,1,2。

## 常见错误排查

### ❌ 错误1：上传后显示"0行0列"
**原因**：文件编码问题或格式不兼容  
**解决**：
1. 确认文件是 UTF-8 编码（COMSOL 默认）
2. 检查是否有中文路径（系统暂不支持）
3. 用记事本打开，确认是纯文本格式

### ❌ 错误2：处理时报错"维度不匹配"
**原因**：坐标列数设置错误  
**解决**：
1. 检查数据文件的 `% Dimension` 值
2. 2D轴对称 → 坐标列数=2，3D笛卡尔 → 坐标列数=3
3. 重新创建数据集（删除旧的）

### ❌ 错误3：perfile 模式工况值识别错误
**原因**：文件名不符合规范  
**解决**：
1. 文件名必须以数字开头（如 `100.txt`, `100[A].txt`）
2. 或上传后手动设置 conditionValue
3. 批量重命名工具：使用上面的 Python 脚本

### ❌ 错误4：PCA降维后维度太小
**原因**：样本数量 < 主成分数  
**解决**：
1. 检查训练集样本数（如只有20个工况 → 最多19个主成分）
2. 减少 PCA 主成分数设置
