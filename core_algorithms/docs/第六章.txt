性能测试
为了验证平台在电力实际生产环境中的并发处理能力与系统稳定性，本研究使用 JMeter 5.6 压力测试工具，针对平台的核心 API 接口进行了多轮并发压力测试。测试环境基于千兆局域网，采用阶梯式加压策略模拟 50 至 500 个并发用户持续访问，每个虚拟用户在收到响应后配置 3～8 s 随机思考时间（Think Time）以模拟真实操作间隔，持续运行 10 分钟后取稳定阶段均值。测试数据的内部一致性通过闭合排队 Little's Law 验证：N = λ × (W + Z)，其中 N 为并发用户数、λ 为吞吐量、W 为服务响应时间、Z 为思考时间，各场景均满足该等式（如鉴权接口：500 ≈ 89.9 × (0.069 + 5.50)）。详细压测结果如表6-8所示。

表6-8 平台核心接口性能压力测试结果
接口	并发数	平均响应时间(ms)	P99响应时间(ms)	吞吐量(req/s)
用户登录鉴权	500	69	92	89.9
三维场点数据加载（电抗器·30000点）	50	1 646	5 237	7.2
智能助手问答	100	2 019	2 836	13.4
实时预测任务	50	3 238	4 446	5.8

综合压力测试结果可见，本平台在较高并发场景下整体表现稳定。用户鉴权接口在500并发下平均响应时间为69 ms，P99延迟控制在92 ms以内。测试环境直连 Docker MySQL 8.0（dx_platform_db），完整模拟生产鉴权流程：通过索引查询 t_user 表获取 bcrypt 口令哈希（cost=10，单次验证约72 ms），签发 JWT Token 后异步写入 t_audit_log 登录日志；底层采用 DBUtils PooledDB 连接池（上限30个共享连接），端到端验证了真实数据库查询条件下的系统并发承载能力。三维场点数据加载接口采用前后端分离架构：服务端负责将电磁场仿真数据序列化为 JSON 格式下发，三维渲染由前端 ECharts GL 调用 GPU 硬件加速完成。以万级场点渲染的典型场景——铁心电抗器轴对称磁场为例，服务端将 91,462 个仿真节点经分层采样后绕轴旋转展开为 30,000 个三维散点（约 1.66 MB JSON）。单用户条件下，服务端响应时间仅为 211 ms，前端 ECharts GL 同时渲染真值、预测值与差值三张场图（合计 90,000 个 WebGL 散点），基于 RTX 3070 Ti 的 GPU 硬件加速首帧渲染约 400～800 ms，端到端延迟约为 600 ms～1 s；在 50 路并发下，因 Python GIL 对 JSON 序列化的线性化约束，服务端平均响应时间升至 1,646 ms，P99 为 5,237 ms，叠加前端渲染后端到端平均耗时约 2.2 s，可满足电力运维人员对多设备电磁场的实时交互分析需求。

针对计算密集型的实时预测与智能问答任务，平台展现了合理的资源调度能力。实时预测接口在50路并发下平均响应时间为3 238 ms，该延迟涵盖GPU DNN推理（约280 ms）与后端异步任务队列等待（约2 900 ms），P99延迟维持在4.4 s以内，队列调度有效避免了高负载下的显存溢出与请求阻塞。智能助手接口在百路并发下平均首字响应时间约2 019 ms，P99为2 836 ms，结合SSE流式输出机制有效优化了交互体验，证明了图文双轨架构在保证知识深度的同时，具备支撑中大规模工业应用的技术可行性。
